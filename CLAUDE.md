## Architecture

Single-page static site. No SSR, no backend.

### Tech Stack

- **Vite** — bundler and dev server (via `vite-plus`)
- **React 19** — UI
- **TanStack Router** — file-based routing (`src/routes/`)
- **Bun** — package manager
- **Vite+** (`vite-plus`) — linting and formatting (Oxlint + Oxfmt)
- **TypeScript** — via `tsgo` (`@typescript/native-preview` compatible `typescript` package)
- **Tailwind CSS v4** — utility-first CSS via `@tailwindcss/vite` plugin. Theme in `@theme` block in `styles.css`

### Project Structure

```
src/
  main.tsx              # App entry — creates router, renders to #root
  styles.css            # Tailwind imports, @theme, and custom CSS (pseudo-elements, animations)
  routeTree.gen.ts      # Auto-generated by TanStack Router plugin
  routes/
    __root.tsx           # Root layout (imports styles, renders <Outlet>)
    index.tsx            # Homepage — renders LandingPage component
  components/
    landing-page/        # Landing page layout, nav, tabs, footer
      cast/              # Cast package sections (hero, features, dx, more, showcase, quickstart, trust-bar)
      prospect/          # Prospect package sections (hero, features, showcase, quickstart, trust-bar)
      constants.ts       # Shared code snippet constants
    copy-button.tsx      # Reusable copy-to-clipboard button
    pm-switcher.tsx      # Package manager tab switcher (bun/npm/yarn/pnpm)
  lib/
    pkg-config.ts        # Package URLs, PM commands, type definitions
```

### Deployment

GitHub Pages via `.github/workflows/deploy.yml`. On push to `main`:
1. `bun install --frozen-lockfile`
2. `bun run build`
3. Push `dist/` to `ingotjs/ingotjs.github.io` (main branch) via `peaceiris/actions-gh-pages@v4`

**CI secret:** `SITE_DEPLOY_TOKEN` (GitHub personal access token with repo access to `ingotjs/ingotjs.github.io`)

### Linting & Formatting

[Vite+](https://vite.dev/plus/) (`vite-plus`) for Oxlint + Oxfmt. Config: `vite.config.ts` imports from `vite-plus`.

- **Format**: `vp fmt --write`
- **Lint**: `vp lint --fix`
- **Do NOT install Oxlint, Oxfmt, or tsdown directly** — Vite+ bundles them

## Development Rules

### Code Standards

**Clean Code + KISS + YAGNI** — self-documenting, readable, minimal complexity.

### File Size

- **Non-autogenerated source files MUST stay under ~300 lines.** Small tolerance is fine, but if a file grows past this, split it. Create a directory in the same location as the file to house the split modules (e.g., `foo.tsx` -> `foo/bar.tsx` + `foo/baz.tsx`).
- **Keep functions and components short.** Extract helpers, sub-components, and constants. Avoid long inlined JSX, objects, or logic blocks — break them into well-named pieces.

### TypeScript

- **NEVER** `any`, `as any`, `interface`, or `catch (error: any)`
- MUST reuse existing types
- Prefer optional chaining for callbacks: `onComplete?.(data)`
- Object params over positional: `function foo({ name }: { name: string })`

### Imports

- **NEVER use barrel files** (index.ts re-exports)
- **MUST import directly from source files**
- **NEVER use dynamic imports** unless genuine code splitting

### Comments

- NEVER add "what I changed" comments
- Only comment complex, non-obvious logic
- **MUST add reference links:** `// Reference: https://...`

### React & Frontend

- **Mobile-first responsive design — NON-NEGOTIABLE**
- Rules of Hooks (top level only)
- Function components only
- Semantic HTML and ARIA attributes for accessibility
- `rel="noopener"` on `target="_blank"` links

### Debugging

- NEVER assume the cause — add targeted debugging
- Trace data flow backwards from the error
- `console.log('DEBUG:', ...)` — MUST clean up after
- MUST try solutions before suggesting them

### Implementation

- MUST implement FULLY — NEVER leave "to be implemented" placeholders
- NEVER create documentation files unless explicitly requested

## Git Workflow

- **NEVER commit or push unless explicitly instructed.** Show changes, wait for instruction. **NEVER commit/push automatically as part of a workflow.**
- `"commit"` = commit EVERYTHING + push. `git add -A && git commit -m "..." && git push`
- `"commit staged only"` = commit only staged files + push
- **Review changes:** `git diff HEAD --stat` for summary, `git diff HEAD -- '*.ts' '*.tsx' '*.css' '*.json' ':!bun.lock'` for code. NEVER read unfiltered diff.
- New branches: `git fetch origin && git checkout -b <name> origin/main` (always from remote)
- First push: `git push -u origin <branch-name>`
- GitHub ops: ALWAYS use `gh` CLI
- **When committing: MUST update CLAUDE.md** if anything documented here changed
- **NEVER add Co-Authored-By or any AI attribution to commit messages**
- **NEVER use `--no-verify`** unless the user strictly says "skip hooks" or "no-verify". If the hook fails, FIX the issue.
